Dсе классы, которые будут работать с БД будут находиться в данной директории (dao: data access objects).
Нужно создать интерфейс (для взаимодействия конкретного интерфеса с БД - юзер, роль и т.п., прим, UserDao).
В интерфейсе прописываем нужные методы: CRUD operations:
- Create=Insert
- Read=Select
- Update
- Delete
На примере UserDao прописываем findAll. По сути это Read.

Для данного интерфейса мы создаем класс-реализацию (прим. UserDaoImpl).
Для каждой таблицы, которая у нас есть в БД создаем класс-аналог, который будет содержать все поля, которые соответствуют колонкам в БД.
Классы-аналоги создаем в папке domain.
